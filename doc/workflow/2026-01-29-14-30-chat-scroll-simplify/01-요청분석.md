# 요청 분석: 채팅 스크롤 기능 단순화 (Option A)

## 요청 개요
이전 워크플로우에서 실패한 채팅 스크롤 기능 구현을 Option A(요구사항 범위 축소) 방식으로 해결

## 이전 실패 원인
| 심각도 | 이슈 |
|--------|------|
| CRITICAL | ChatContainer와 ChatView 간 ref 타입 계약 불일치 |
| HIGH | forwardRef 제네릭 타입 union 문제 |
| HIGH | 단일 return 구조에서 ref 배치 오류 |

## Option A 해결 방향
1. forwardRef 패턴 완전 제거
2. 컴포넌트 간 ref 전달 제거
3. MessageListView 내부에서 자체적으로 스크롤 관리
4. 메시지가 있을 때만 스크롤 동작 (빈 상태 제외)

## 수정 대상 파일
- `src/features/chat/ChatContainer.tsx`
- `src/features/chat/ChatView.tsx`
- `src/features/chat/MessageListView.tsx`

## 기대 효과
- 타입 복잡성 제거
- 컴포넌트 간 결합도 감소
- 스크롤 로직 단순화 및 유지보수성 향상
