# 요청 분석: 채팅 메시지 동적 스크롤 기능 개선

## 요청 내용
사용자가 채팅 입력 후 결과를 받으면 동적으로 스크롤이 이동되도록 개선 요청

## 현재 상태 분석

### 기존 구현
- **파일**: `src/features/chat/ChatContainer.tsx`
- **구현 방식**:
  ```tsx
  const messagesEndRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    messagesEndRef.current?.scrollIntoView({ behavior: 'smooth' });
  }, [messages]);

  return (
    <>
      <ChatView ... />
      <div ref={messagesEndRef} />
    </>
  );
  ```

### 문제점
1. `messagesEndRef`가 `ChatView` 컴포넌트 **외부**에 위치
2. 실제 스크롤 영역은 `MessageListView` 내부의 `overflow-y-auto` div
3. 스크롤 타겟과 스크롤 컨테이너가 분리되어 있어 `scrollIntoView`가 의도대로 동작하지 않음

### 관련 컴포넌트 구조
```
ChatContainer
├── ChatView
│   ├── Header
│   ├── MessageListView (overflow-y-auto) ← 실제 스크롤 컨테이너
│   │   └── messages.map(...)
│   └── MessageInputView
└── <div ref={messagesEndRef} /> ← 스크롤 타겟 (잘못된 위치)
```

## 작업 범위

### 필요한 작업
- **Frontend**: 스크롤 ref 위치 수정 및 로직 개선
- **Backend**: 해당 없음 (순수 클라이언트 사이드 UI 작업)

### 수정 대상 파일
1. `src/features/chat/MessageListView.tsx`
2. `src/features/chat/ChatContainer.tsx`
3. 필요시 `src/features/chat/ChatView.tsx`
