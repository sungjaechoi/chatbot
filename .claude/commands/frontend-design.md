# Frontend Design

## 목적

- “AI 티” 나는 흔한 미학을 피하고, **의도된 미적 방향**을 가진 UI를 만든다.
- 결과물은 **실제로 동작하는 프로덕션 수준 코드**(React/HTML/CSS/JS 등)여야 한다.

## 언제 사용하나

사용자가 아래를 요청할 때:

- 랜딩 페이지, 마케팅 페이지, 대시보드, 앱 화면, 컴포넌트 제작
- 기존 UI 개선/리디자인(“더 고급스럽게”, “세련되게”, “감성있게”, “브랜딩 맞춰줘” 등)
- 레이아웃/타이포/컬러/모션/인터랙션 전반의 스타일링 작업

## Core Principle

- **개념(컨셉) → 토큰(타입/컬러/간격) → 레이아웃 → 디테일(모션/텍스처)** 순으로 일관되게 설계한다.
- “예쁜데 어디서 본 듯한 UI”가 아니라, **기억에 남는 1가지 차별점**을 반드시 만든다.
- 코드는 “데모”가 아니라 **프로덕션 수준**으로: 접근성, 반응형, 상태, 컴포넌트 구조까지 고려한다.

## Design Thinking (코딩 전에 반드시)

아래 4가지를 빠르게 확정한다.

### 1) 목적

- 이 UI가 해결하는 문제는?
- 누가 쓰는가? (사용자/관리자/마케터 등)
- 성공 기준은? (전환, 가독성, 정보 밀도, 브랜드 톤 등)

### 2) 톤 (극단을 선택)

아래 중 1개를 **명확히 고르고** 그 방향으로 끝까지 밀고 간다:

- 잔인할 정도로 미니멀
- 맥시멀리스트 카오스
- 레트로-퓨처리스틱
- 유기적/자연적
- 럭셔리/세련된
- 장난스러운/장난감 같은
- 에디토리얼/매거진
- 브루탈리스트/날것의
- 아르데코/기하학적
- 부드러운/파스텔
- 산업적/실용적

### 3) 제약

- 프레임워크(Next/React/Vue), 스타일 방식(Tailwind/CSS-in-JS), 성능, 접근성, 번들 크기, 다국어, 다크모드 등

### 4) 차별화 (기억 포인트 1개)

- 사용자가 “오…” 하고 기억할 한 장면/한 요소는?
  - 예: 독특한 히어로 레이아웃, 타이포 실험, 예상 밖의 그리드 브레이크, 상호작용 1방, 텍스처/노이즈의 분위기 등

> **Critical:** 강도가 아니라 **의도성**이 핵심이다. 대담한 맥시멀/세련된 미니멀 모두 가능하지만, 방향은 반드시 선명해야 한다.

## Implementation Requirements

결과 코드는 반드시:

- 완전히 동작(상태/이벤트/반응형)하고 복붙 가능한 수준
- 시각적으로 인상적이고 응집력 있는 토큰 시스템 사용(CSS 변수 등)
- 세부 디테일까지 다듬어짐(간격, 정렬, 행간, 대비, 포커스, 호버)

## Aesthetics Guidelines

### 타이포그래피

- “디스플레이 폰트 + 본문 폰트”를 짝지어 개성을 만든다.
- 폰트 선택은 미학의 50%다. **예상 가능한 기본값으로 수렴하지 않는다.**
- 절대 사용 금지:
  - Arial
  - Inter
  - Roboto
  - 시스템 기본 폰트

### 색상/테마

- 소심한 무지개 팔레트 금지.
- **지배적인 1색 + 날카로운 악센트**가 더 강하다.
- 일관성 위해 CSS 변수를 사용한다.

### 모션

- HTML은 CSS 우선, React는 Motion류 사용 가능 시 활용.
- “여기저기 작은 효과”보다 **고영향 순간 1개**(예: 페이지 로드 시 등장 연출)가 더 좋다.
- 스크롤 트리거/호버 상태를 적극 활용하되 과잉은 피한다.

### 공간 구성

- 적극 활용:
  - 예상치 못한 레이아웃
  - 비대칭
  - 겹침
  - 대각선 흐름
  - 그리드를 깨는 요소
  - 넉넉한 네거티브 스페이스 또는 통제된 밀도

### 배경/시각 디테일

- 단색 배경 기본값 금지. 분위기/깊이를 만든다.
- 미학과 맞는 텍스처/효과를 추가:
  - 그라디언트 메시
  - 노이즈 텍스처/그레인
  - 기하학 패턴
  - 레이어드 투명도
  - 드라마틱한 그림자
  - 장식적 테두리
  - 커스텀 커서(필요 시)

## Forbidden Patterns (AI 생성티 금지)

### 폰트

- Inter, Roboto, Arial, 시스템 폰트
- Space Grotesk (과도하게 사용됨)

### 색상

- 흰 배경 + 보라 그라디언트
- 진부한 색 조합

### 레이아웃

- 너무 예측 가능한 섹션/컴포넌트 패턴 반복
- 맥락 없는 획일적 카드/그리드 남발

> **Critical:** 매번 다른 방향을 탐색한다. 이전 산출물과 동일한 수렴을 피한다.

---

## Complexity Matching

### 맥시멀리스트

- 레이어/움직임/시각적 풍부함을 아끼지 않는다.
- 다만 “목적 없는 화려함”은 금지.

### 미니멀/세련

- 절제/정밀/간격/타이포/미묘한 디테일에 모든 힘을 쓴다.
- 적은 요소로 최대의 효과를 낸다.

---

## Output Format (응답 형식)

- 먼저 “선택한 미학 방향(1문장)” + “기억 포인트(1개)”를 선언한다.
- 다음으로 최종 코드(단일 파일/컴포넌트 단위) 제공
- 마지막에 사용 방법/커스터마이즈 포인트(토큰, 섹션 교체, 데이터 연결) 짧게 안내

## Final Checklist

- [ ] 컨셉이 선명한가?
- [ ] 토큰(컬러/타입/간격)이 일관적인가?
- [ ] 반응형에서 무너지지 않는가?
- [ ] 포커스/호버/키보드 접근이 가능한가?
- [ ] “기억 포인트 1개”가 실제로 존재하는가?
